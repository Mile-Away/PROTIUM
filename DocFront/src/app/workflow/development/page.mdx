# 开发者手册｜开发一个自定义 Node

## 构建工作流时发生的事情


一个工作流由若干个 Node 和 Node 之间相互连接的 Edge 组成，

### 每个 Node 有以下几个部分：

Node 状态，

Node Data，Node Data 中包括

* Node Handle：控制 Node 的输入输出
* Node Body: 记录用户输入的信息
* Node Result：记录 Node 的执行逻辑以及执行结果
* Node Error: 记录 Node 的错误信息

### 每个 Edge 有以下几个部分：

source: Node, 出发点的 Node
target: Node, 目标 Node
sourcehandle: Handle UUID, 出发点的 Node 被连接的 Handle
targethandle: Handle UUID, 目标 Node 被连接的 Handle

### 构建 Node 时的规则

1. Node 左侧的 Handle 用于接收 Edge 连接，右侧的 Handle 用于连接到下一个 Node。
2. handle 同一时间只能连接一个 Edge，当一个新的 Edge 连接到 handle 时，原有的 Edge 会被删除。

### 最佳实践

1. 从左往右构建 Node


## 运行工作流时发生的事情

1. 检查所有工作流中提交的 Node，并返回不同的状态：
    * 所有 Handle 都未被连接，返回状态：skipped
    * 任一 Source Handle 或所有 Target Handle 被连接，返回状态：running
    * 其它任何情况，例如 target handle 未被连接，返回状态：failed

2. 建立 Node 之间的依赖关系，从最终 Node 开始，逐步向前检查，直到遍历所有的 Node。
如果一个 Node A 的 Source Handle 连接到 Node B 的 Target Handle，那么 Node A 是 Node B 的依赖。

3. 从没有依赖的 Node 开始，异步并行执行 Node 中的 Result，直到所有的 Node 都执行完毕。返回状态 Success

4. Node 状态：
* 绿色边框：运行完成，
* 黄色边框：正在运行，
* 红色边框：运行失败，
* Node 变透明：此次运行中跳过。
